Summary

This sample project demonstrates basic usage of the BIOPAC Hardware API in C/C++.

Introduction

There are three functions implemented in this sample program. Each one is designed to show how the most common Hardware API methods can be used in C/C++. The functions are getBufferDemo(), startAcqDaemonDemo(), and ioDemo().

getBufferDemo()

This function acquires 100 samples on analog channels 1 and 5 at 1000 Hz and outputs the data to standard output. It uses the API's data download method getMPBuffer(). Although getMPBuffer() is a blocking function, the acquisition length is short enough that data processing can be done after the function returns without significant delay. When the function returns, the data must be de-interleaved to separate data from channel 1 and channel 5.

startAcqDaemonDemo()

This function acquires 5000 samples on analog channels 2, 7, and 11 at 200 Hz and outputs the data to standard output. Unlike the getBufferDemo() function, it uses the API functions startAcqDaemonDemo() and receiveMPData() to allow real-time data acquisition and processing.

ioDemo()

This function illustrates how to output to the Analog Output and Digital I/O of the MP device using the Hardware API. Hardware API methods that control input and output can be used anytime after a successful call to connectMPDev() and before a call to disconnectMPDev().

Notes

  • Calls to the API method connectMPDev() need to be modified to reflect your MP device configuration
  • File mpdev.lib needs to be referenced or moved to the project directory
  • Path to mpdev.h needs to be specified in the include statement or move to the project directory
